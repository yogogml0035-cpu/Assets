# Assets

这是一个面向 AI Agent、上下文工程、提示词和交易分析的 Markdown 知识资产库。仓库以主题目录组织资料，优先保持原始资料的来源信息、标题语义和可追溯性。

| 目录 | 内容边界 |

| [Harness工程](Harness工程/) | Agent harness、framework 边界和工程稳定性相关资料。 |

## 推荐阅读路径

- 学习 DeepAgents：先读 [DeepAgents/README.md](DeepAgents/README.md)，再按（一）到（七）顺序阅读系列文档。
- 梳理 Agent 工程边界：先看 `Harness工程/`，再看 `多Agent工程/` 和 `上下文工程/` 中的架构拆解。
- 复用提示词：从 `常用提示词/` 选择对应场景，必要时再参考 `历史长需求记录/` 中的长任务样例。

## 分类与维护规则

1. 新增文档时，优先放入最具体的主题目录；无法确定归属时先放入 `Clippings/`，并在后续整理时迁移。
2. 同一来源或同一主题的多份文档应指定一份主文档，并在相关副本文档顶部说明用途和主文档链接。
3. 文件名应反映真实内容。交易分析文档需要包含日期、标的和主题；系列文章需要保留序号。
4. 保留剪藏文档的 front matter、来源链接、作者和创建时间等溯源信息。
5. 修改提示词模板时，避免只改局部措辞却改变适用场景；如果模板职责变化，应同步调整标题或新增独立文档。

## 验证、发布与安全说明

- 当前仓库主要保存 Markdown 文档，没有应用代码、依赖清单或构建产物。
- 文档修复的基础验证为 `git diff --check`，必要时结合定向搜索确认标题、链接、错字或代码块格式。
- 当前没有独立发布流程；合并到默认分支即代表知识库内容更新。
- 当前没有运行时安全边界或依赖漏洞审计对象。若后续加入脚本、示例项目或自动化任务，应补充对应测试、依赖审计和安全说明。
